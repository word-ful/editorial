# Contributing

These skills are open source and we welcome contributions — patterns we missed, rules that misfire, examples that sharpen the spec.

## Reporting issues

If a skill missed something or false-flagged something, open an issue with:

- The passage in question (anonymize if needed)
- The rule that should have fired, or that fired incorrectly
- A short note on what you expected

## Proposing rule changes

The reference ruleset for `dissolve-ai` lives at [`dissolve-ai/reference/patterns.md`](./dissolve-ai/reference/patterns.md). Improvements welcome via PR. Please include:

- The pattern being added, modified, or removed
- 2–3 real-world examples (anonymized if needed)
- A short rationale for the change

## Pull requests

Keep them focused. One pattern change per PR is easier to review than a sweep. Reference the related issue if there is one.

## A note on scope

These skills are descriptive, not prescriptive. They identify tendencies that undermine authentic voice — they are not grammar rules, and they are not absolute. Contributions that sharpen the diagnostic value of a rule are more useful than contributions that expand the rule into territory it was not designed for.
